# Expert Skills

AIをMunger、Feynman、Paul Grahamのように考えさせる ── ロールプレイではなく、彼らが実際に意思決定に使うメンタルモデルをロードすることで。

```
あなた：「友人がレストランを開くので500万円出資してほしいと。料理の腕は確かです。」

Claude：「なるほど、その投資機会について見てみましょう...」

Claude + expert-munger：
        「『料理の腕は確か』とのことですが、これは能力の輪の逸脱
        （Circle of Competence Violation）です。
         料理のスキルで投資判断をしています。
         逆転の発想：5年後にこの投資が失敗するとしたら、最も
         ありそうな理由は何でしょう？
         貯蓄の33%、換金不可、根拠は人柄だけです。」
```

> Claudeに人格を与える（「チャーリー・マンガーとして振る舞え」）よりも、マンガーの実際のフレームワークをロードする方が優れた出力を生みます。人格は装飾。フレームワークが本体です。

---

## インストール

```bash
npx openskills install https://github.com/AskRoundtable/expert-skills
```

Claude Code、Cursor、Windsurf、その他SKILL.md互換ツールで動作します。

## エキスパート

| モジュール | 思考法 | こんな質問に |
|-----------|--------|------------|
| **expert-munger** | 逆転思考、能力の輪、安全域 | 「この投資は妥当？」「この判断の最大のリスクは？」 |
| **expert-naval** | レバレッジ（コード/メディア/資本）、固有知識、Principal vs Agent（当事者 vs 代理人） | 「この仕事を受けるべき？」「長期的な富をどう築く？」 |
| **expert-feynman** | 第一原理分解、ファインマン・テクニック、アナロジー | 「小学生にもわかるようにXを説明して」「これは実際どう動く？」 |
| **expert-kahneman** | システム1 vs システム2、損失回避、アンカリング | 「私の判断は合理的？」「どんなバイアスを見落としてる？」 |
| **expert-graham** | スケールしないことをやれ、シュレップ盲点、Default Alive | 「このスタートアップのアイデアはいい？」「最初のユーザーをどう獲得する？」 |

## 使用例

**キャリア判断 + expert-naval：**
```
あなた：「オファーが来ました。年収40%アップですが、コーディングからマネジメントに移ります。」

Claude：「大幅な昇給ですね。メリットとデメリットを整理しましょう...」

Claude + expert-naval：
        「これはレバレッジの罠です。許可不要のレバレッジ（コード）を、
         最も古く弱い形態 ── 労働力レバレッジ（人の管理）に交換しようと
         しています。昇給は線形ですが、コーディング力は複利で成長します。
         問うべきは：10年後、どちらの自分がより多くの固有知識と
         選択肢を持っているか？」
```

**スタートアップのアイデア + expert-graham：**
```
あなた：「AIを使った献立アプリを作りたい。市場は巨大です。」

Claude：「素晴らしいアイデアですね！献立市場は急成長中で...」

Claude + expert-graham：
        「『市場は巨大』── これはタールピット・アイデアのシグナルです。
         明らかに良いアイデアで市場も大きいなら、なぜまだ誰も勝っていない？
         みんなが避けているシュレップ（面倒な仕事）は何か？
         もっと重要なのは：あなた自身がこの問題を抱えているか？
         これを作ってDefault Alive（資金なしで存続可能）か、
         それとも調達に賭けているか？」
```

**認知バイアス + expert-kahneman：**
```
あなた：「3つのマーケティング戦略をテストしました。戦略Bが先週
         クリック率20%高かったので、全力投入します。」

Claude：「戦略Bが明らかに優れています。スケールの方法は...」

Claude + expert-kahneman：
        「1週間のデータで結論を出すのは、典型的なシステム1の思考です。
         脳がノイズから物語を作りたがっています。3戦略・1週間では、
         小さなサンプルにアンカリングされています。
         クリックデータの週次変動の基準率は？
         これは勝者の衣を纏った平均回帰かもしれません。」
```

## なぜ「マンガーとして振る舞え」ではダメなのか？

ペルソナプロンプトはAIに「誰になるか」を伝えます。Expert Skillsは「どう考えるか」を伝えます。

各モジュールはエキスパートの実際の認知プロセスをエンコードしています：
- **パターン**：認識する状況（「これは能力の輪の逸脱に見える」）
- **メンタルモデル**：適用するフレームワーク（逆転思考、レバレッジ、第一原理）
- **手がかり感度**：他の人が見落とす細部への気づき
- **盲点**：自身の限界（分析が自らの弱点を指摘する）

すべてを6ステップの認知エンジンが駆動。実際のエキスパートが不確実性の下でどう意思決定するかに基づいています（Gary KleinのRPDモデル）。

## パフォーマンス

ベースラインClaude（エキスパートモジュールなし、temperature=0）との比較。4次元の評価基準で、エキスパートごとに3つの実例で評価。

| エキスパート | 領域 | ベースライン比 |
|-------------|------|--------------|
| expert-graham | スタートアップ / グロース | +38%〜+83% |
| expert-feynman | 第一原理 / 学習 | +38%〜+57% |
| expert-naval | キャリア / 富 / 人生 | +38%〜+57% |
| expert-kahneman | 行動経済学 / 意思決定 | +31%〜+63% |
| expert-munger | 投資 / ビジネス | +18%〜+41% |

評価次元：パターン認識精度、メンタルモデル適用、盲点認識、推奨の実行可能性。

## 仕組み

二層アーキテクチャ：認知エンジンが思考プロセスを実行し、エキスパートモジュールが領域知識を提供。

```
┌──────────────────────────────────────────┐
│              expert-engine               │
│          6ステップ認知プロセス               │
│                                          │
│  パターン認識 → メンタルシミュレーション       │
│  → 異常検知 → インサイト                    │
│  → 認識論的監査                            │
└──────────────────┬───────────────────────┘
                   │ 以下から領域コンテンツをロード
        ┌──────────┼──────────┐
        ▼          ▼          ▼
  expert-munger  expert-naval  expert-feynman
  patterns.md    models-core   blind-spots.md
  models-core    blind-spots   ...
```

エンジンは毎回実行されます。エキスパートモジュールが状況パターン、メンタルモデル、手がかり感度リスト、既知の盲点を提供 ── 領域知識なしにはエンジンが持ち得ないコンテンツです。
