# Expert Skills

讓你的 AI 像 Munger、Feynman 或 Paul Graham 一樣思考 —— 不是角色扮演，而是載入他們實際用來做決策的心智模型。

```
你：    「朋友想找我投資 150 萬開餐廳，他廚藝很好。」

Claude：「好的，我們來看看這個機會⋯⋯」

Claude + expert-munger：
        「你說『他廚藝很好』—— 這是能力圈謬誤（Circle of Competence Violation）：
         你用廚藝來判斷投資能力。
         反向思考：如果 5 年後這筆投資失敗，最可能的原因是什麼？
         佔你 33% 的存款、無法變現，而你唯一的證據是一個人格特質。」
```

> 給 Claude 一個人設（「扮演查理蒙格」）的效果，比載入蒙格的實際框架更差。人設是裝飾，框架才是機制。

## 安裝

```bash
npx openskills install https://github.com/AskRoundtable/expert-skills
```

支援 Claude Code、Cursor、Windsurf，或任何相容 SKILL.md 的工具。

## 專家

| 模組 | 思考方式 | 可以問他們 |
|------|---------|-----------|
| **expert-munger** | 反向思考、能力圈、安全邊際 | 「這筆投資值得嗎？」「這個決定最大的風險是什麼？」 |
| **expert-naval** | 槓桿（程式/媒體/資本）、特定知識、擁有者 vs 執行者 | 「該不該接這個工作？」「怎麼建立長期財富？」 |
| **expert-feynman** | 第一性原理拆解、費曼技巧、類比 | 「用小學生聽得懂的方式解釋 X」「這東西到底怎麼運作的？」 |
| **expert-kahneman** | 系統一 vs 系統二、損失趨避、錨定效應 | 「我的判斷理性嗎？」「我可能有什麼偏誤？」 |
| **expert-graham** | 做不規模化的事、苦差事盲點、預設存活 | 「這個創業點子好嗎？」「怎麼找到前十個用戶？」 |

## 範例

**職涯決策 + expert-naval：**
```
你：    「收到一個 offer，加薪 40%，但要從寫程式轉去帶團隊。」

Claude：「加薪幅度很大，我們來分析利弊⋯⋯」

Claude + expert-naval：
        「這是一個槓桿陷阱。你正在把無需許可的槓桿（程式碼）
         換成最古老、最弱的形式 —— 勞動力槓桿（管人）。
         加薪是線性的，但你的程式能力是複利成長。
         問自己：10 年後，哪個版本的你擁有更多特定知識和更多選擇？」
```

**創業點子 + expert-graham：**
```
你：    「我想做一個 AI 餐點規劃 app，市場很大。」

Claude：「好主意！餐點規劃市場正在快速成長⋯⋯」

Claude + expert-graham：
        「你說『市場很大』—— 這是焦油坑點子（Tarpit Idea）的信號。
         如果這麼明顯的好點子、市場又大，為什麼還沒有人贏？
         大家在迴避什麼苦差事（schlep）？
         更重要的是：你自己有這個問題嗎？做這個你是預設存活（Default Alive）
         還是在賭融資？」
```

**認知偏誤 + expert-kahneman：**
```
你：    「我們測了 3 個行銷策略，策略 B 上週點擊率高了 20%，
         決定全力押注策略 B。」

Claude：「策略 B 明顯表現最好，以下是擴大規模的方法⋯⋯」

Claude + expert-kahneman：
        「一週的數據就下結論，這是典型的系統一思考 —— 你的大腦想從
         雜訊中編出一個故事。3 個策略、1 週時間，你被錨定在一個小樣本上。
         你們點擊數據的週間基準波動率是多少？
         這可能只是均值回歸穿了贏家的外衣。」
```

## 為什麼不直接說「扮演蒙格」？

人設提示告訴 AI「成為誰」。Expert Skills 告訴它「怎麼思考」。

每個模組編碼了一位專家的實際認知過程：
- **模式**：他們辨識出的情境（「這看起來是能力圈謬誤」）
- **心智模型**：他們套用的框架（反向思考、槓桿、第一性原理）
- **線索敏感度**：他們注意到而別人忽略的細節
- **盲點**：他們自身的局限（所以分析會標記自己的弱點）

全部由一個六步驟認知引擎驅動，基於真實專家在不確定情境下如何做決策（Gary Klein 的 RPD 模型）。

## 效能

以基線 Claude（無專家模組、temperature=0）為對照，使用四維度評分標準，每位專家測試 3 個真實案例。

| 專家 | 領域 | 相較基線提升 |
|------|------|-------------|
| expert-graham | 新創 / 成長 | +38% 至 +83% |
| expert-feynman | 第一性原理 / 學習 | +38% 至 +57% |
| expert-naval | 職涯 / 財富 / 人生 | +38% 至 +57% |
| expert-kahneman | 行為經濟學 / 決策 | +31% 至 +63% |
| expert-munger | 投資 / 商業 | +18% 至 +41% |

評分維度：模式辨識準確度、心智模型應用、盲點意識、建議可行性。

## 運作原理

雙層架構：認知引擎負責思考流程，專家模組提供領域知識。

```
┌──────────────────────────────────────────┐
│              expert-engine               │
│           六步驟認知流程                    │
│                                          │
│  模式辨識 → 心智模擬                        │
│  → 異常偵測 → 洞察                         │
│  → 認知審計                               │
└──────────────────┬───────────────────────┘
                   │ 從以下載入領域內容
        ┌──────────┼──────────┐
        ▼          ▼          ▼
  expert-munger  expert-naval  expert-feynman
  patterns.md    models-core   blind-spots.md
  models-core    blind-spots   ...
```

引擎每次都會執行。專家模組提供情境模式、心智模型、線索敏感度清單和已知盲點 —— 這些是引擎沒有領域知識就無法擁有的內容。
